# 基金定投策略回测工具

一个本地桌面 GUI 程序，可以让你自由配置 "什么时候买、什么时候卖、买/卖多少" 这种自定义规则，然后用 **akshare** 拉取真实开放式基金的历史净值，在 1月 / 3月 / 6月 / 1年 / 3年 五个区间内分别回测出总收益，画出策略价值曲线、基金净值曲线，并自动标出"理论上最适合卖出的几天"。

适合的使用场景：

- 你心里有一套"跌2%买50、跌3%买100、跌>4%买200"或者"每天定投30、每周一定投100"这样的策略，想看在某只基金上跑出来到底赚不赚钱。
- 你想比较同一只基金下，多套策略（多个预设）哪个表现更好。
- 你想横向看几只不同的基金（恒生科技、A500、纳指、消费 ETF…）在你的策略下的差距。

## 主要功能

### 1. 基金数据管理
- 输入 6 位基金代码 → 点 "获取最新数据" 通过 akshare 拉取**全部历史单位净值**，保存到 `data/<code>.csv`。
- 顶部"已保存基金"下拉框记录所有获取过的基金，便于快速切换。
- 旁边自动显示 **"已获得 YYYY-MM-DD ~ YYYY-MM-DD 的数据（共 N 个交易日）"**，让你一眼看清数据范围。
- 同时拉取并缓存基金简称，下拉框显示 `基金简称 (代码)`。

### 2. 策略规则
每条规则包含以下字段：

| 字段 | 含义 |
| --- | --- |
| 名称 | 自定义标签，仅用于显示 |
| 频率 | `daily` 每个交易日 / `weekly` 每周指定星期 / `monthly` 每月指定日期（非交易日顺延到下一个交易日） |
| 星期 | 0=周一 … 6=周日（仅 weekly 生效） |
| 月日 | 1–28（仅 monthly 生效） |
| 方向 | `drop` 跌 / `rise` 涨 / **`any` 不限涨跌（无条件触发）** |
| 阈值% | 日涨/跌幅达到该百分比时触发（`any` 时忽略） |
| 操作 | `buy` 买入 / `sell` 卖出 |
| 金额 | 本次买入或卖出的元数 |

#### 规则匹配语义
- **`any` 方向** = 无条件触发，**多条都会独立执行**。可用来实现"每日/每周/每月定投"。
- **`drop` 或 `rise` 方向** = 条件触发，**同向多条命中时只触发阈值最大的那一条**（自然实现"跌2/3/4% 分档加仓"）。
- "any" 与 "drop/rise" 在同一日都满足时，会**叠加触发**（即既执行无条件定投，也执行触底加仓）。
- 卖出金额自动限制为当前持仓市值，避免超卖。

#### 策略预设管理
- 多套规则保存为不同的 **策略预设**，下拉框切换。
- 默认自带 `默认`（跌2/3/4% 分档加仓）和 `每日定投`（每日 30 元）两个示例。
- 点 "保存策略" 时会弹框让你输入名称：
  - 输入新名 → 另存为新预设并自动切换
  - 输入已有名 → 提示是否覆盖
- 支持重命名、删除预设（至少保留一个）。

### 3. 回测
- 单选 1月 / 3月 / 6月 / 1年 / 3年，一键 "运行回测"。
- 左侧文字汇总：累计买入/卖出、持仓份额、末日净值、持仓市值、盈亏、收益率（= 盈亏/累计买入），以及"等额买入并持有"的基准收益率作为对比。
- 右侧图表：**左轴**为元，画 "策略总价值" 和 "净投入累计" 两条线；**右轴**为基金单位净值。
- **推荐卖出日**：自动找出截至当日累计收益率 > 5% 的最高 3 天（且彼此至少间隔 14 个交易日），在图上用**红色圆点 + 注解**标出日期和收益率，文字结果区也列出对应清单。这里的"收益率"用的是 `(总价值 − 累计买入) / 累计买入`，即截至那天投入的所有本金作为分母。

### 4. 导出
文字结果（左侧三个按钮）：
- **导出当前结果**：把当前显示的文字汇总（含全部交易明细）保存为 `.txt`。
- **导出所有回测结果**：对**当前基金**一次性跑完 5 个区间（1月/3月/6月/1年/3年），写入一个汇总 `.txt`。
- **导出所有基金结果**：用**当前选中的时间区间**，把策略跑遍**所有已保存的基金**，写入一个汇总 `.txt`。

图表（右侧三个按钮）：
- **导出图片**：把当前图表存为 `.png` / `.pdf` / `.svg`。
- **导出所有回测图片**：选一个目录，对**当前基金**的 5 个区间各导出一张图。
- **导出所有基金图片**：选一个目录，用**当前选中的时间区间**，对**所有已保存的基金**各导出一张图。

## 安装与运行

```bash
pip install -r requirements.txt
python app.py
```

> Tkinter 是 Python 内置，无需额外安装。中文图表字体默认使用 Windows 自带的 `Microsoft YaHei`。

## 文件结构

```
app.py              # 主程序（GUI + 回测引擎）
get_HSKJ.py         # 最初参考的数据抓取脚本（可保留可删）
requirements.txt    # 依赖
funds.json          # 已保存基金 {代码: 简称}
strategies.json     # 策略预设 {presets: {name: [rules]}, current: name}
data/<code>.csv     # 各基金的本地净值缓存
data/_all_funds.csv # 基金简称对照表缓存
```

## 数据来源

通过 [akshare](https://akshare.akfamily.xyz/) 的 `fund_open_fund_info_em` 接口获取场外开放式基金的单位净值，`fund_name_em` 获取基金简称。

## 几个常见策略示例

| 想表达 | 怎么写规则 |
| --- | --- |
| 每天无脑定投 30 元 | `daily / any / -- / buy / 30` |
| 每周一定投 200 元 | `weekly / 星期=0 / any / buy / 200` |
| 每月 1 号定投 1000 元 | `monthly / 月日=1 / any / buy / 1000` |
| 跌 2/3/4% 分档加仓 | 三条 `daily / drop / 阈值=2,3,4 / buy / 50,100,200` |
| 涨 5% 卖出 500 元 | `daily / rise / 阈值=5 / sell / 500` |
| 每月 1 号 + 跌幅触发加仓共存 | 把上面规则一起放进同一个预设即可 |
